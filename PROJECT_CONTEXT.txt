Goal:
  Build a RAG-based medical chatbot that retrieves verified medical info,
  generates grounded answers, and supports tool use for report analysis
  and live drug lookups.

Tech Stack (Mandatory):
  - Python
  - FastAPI
  - Qdrant (vector database)
  - Sentence Transformers (embedding)
  - LangChain (agent/orchestration)
  - Llama 3.1 8B via Ollama (LLM)
  - Docker / Docker Compose

Dataset:
  - MedQuAD (Medical Question Answering Dataset)
  - Source: Hugging Face (lavita/medical-qa-datasets or bigbio/med_qa)
  - Size: 16,000+ Q&A pairs
  - Domains: Diseases, symptoms, treatments, drugs, anatomy, genetics

What was defined in Phase 1:

  INGESTION FLOW (offline, one-time):
    1. Load MedQuAD from Hugging Face (datasets library)
    2. Clean text — remove noise, fix formatting, drop empties
    3. Chunk — 300-500 tokens per chunk, 50-token overlap
    4. Embed — Sentence Transformers all-MiniLM-L6-v2 → 384-dim vectors
    5. Store in Qdrant — upsert with deterministic UUID per chunk
       Payload fields: chunk_text, source, category, topic, chunk_index

  INFERENCE FLOW (runs per user request):
    1. User sends message via FastAPI POST /chat
    2. LangChain Agent Router classifies request:
       - Type A: General medical Q&A → RAG path
       - Type B: PDF report upload → Tool 1
       - Type C: Drug name detected → Tool 2
    3. Path A (General Q&A):
       Embed query → Qdrant top-5 search → LangChain prompt → Llama 3.1 8B → answer
    4. Path B (Report Upload — Tool 1):
       Extract PDF text → embed + Qdrant search → send report + context to LLM → plain-language explanation
    5. Path C (Drug Query — Tool 2):
       Detect drug name → scrape drugs.com / FDA → send scraped info + question to LLM → sourced answer
    6. Final answer returned via FastAPI

  TWO TOOLS:
    Tool 1 — PDF Report Analyser
      Trigger: PDF file attached to request
      Input: PDF (lab results, discharge summary, diagnosis)
      Processing: PyMuPDF extraction → embed → Qdrant context → LLM
      Output: Plain-language explanation with medical context

    Tool 2 — Drug Information Retriever
      Trigger: Drug name detected in message
      Input: Drug name (via regex / spaCy NER)
      Data source: drugs.com (primary), FDA (fallback)
      Processing: httpx scrape → BeautifulSoup parse → LLM
      Output: Dosage, side effects, interactions — with source URL cited

  KEY DESIGN DECISIONS:
    - Chunk size 300-500 tokens, 50-token overlap
    - Deterministic point IDs (source + chunk_index hash) for safe re-runs
    - Cosine similarity, top-k=5
    - Same embedding model for ingestion and inference (critical)
    - Stateless inference — no server-side session storage
    - Async FastAPI — non-blocking I/O throughout
